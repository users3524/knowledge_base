## 沉淀建议

- 新增流程文档时，文件名建议采用 `领域 + 软件开发流程` 或 `领域 + 工程规范`。
- 流程类文档尽量包含输入、输出、关键检查点、常用工具链和交付物。
- 如果内容开始偏向具体技术实现，应拆到硬件工程或软件架构对应目录。
