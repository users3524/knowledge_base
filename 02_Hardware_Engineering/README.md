## 收录边界

- 器件共性知识放入 `00_器件知识`，单一型号的临时选型记录不单独建立长期文档。
- 可直接用 EDA 工具打开的参考工程放入 `01_模块整理`，并在后续补充工程版本、用途和验证状态。
- PCB 规则和可复用设计约束放入 `02_PCB设计`，具体项目生产文件仍归档到 `04_Project`。
- 可重复执行的测试步骤放入 `03_调试技巧`；单次故障记录放入其 `事故库`，并保留测试条件、证据、根因和回归结果。
- 能固化为评审动作的内容，提炼后链接到 `05_Templates_&_Checklists`，避免复制出多份长期分叉的 Checklist。

## 文档要求

- 测试文档应写清被测对象、仪器设置、测量点、工况矩阵、合格判据和安全边界。
- 故障复盘应包含问题现象、复现条件、定位过程、根因证据、修改方案、回归结果和预防措施。
- 关键参数应带数值、单位、温度和适用边界；不确定的结论应标明假设或待验证项。
- 新增、移动或重命名文档后，同步更新本页入口和相关相对链接。
