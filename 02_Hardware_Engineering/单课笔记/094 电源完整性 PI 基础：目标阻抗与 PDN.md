## 一、 核心技术边界与 Worst-Case 精算矩阵

本课属于“PCB 叠层、高速 SI 与 PI/DFM”模块，目标不是停留在概念记忆，而是把“电源完整性 PI 基础：目标阻抗与 PDN”放回真实工业硬件开发场景中理解：为什么要这样选型、这样计算、这样画原理图、这样约束 PCB，并最终能用仪器把设计验证闭环。

本课围绕“电源完整性 PI 基础：目标阻抗与 PDN”建立从底层原理、关键参数、Worst-Case 计算、原理图/PCB 约束、仪器验证到量产评审的完整工程闭环。

### 1. 底层概念与核心变量

- **瞬态负载电流阶跃 $dI$**：理解它在「电源完整性 PI 基础：目标阻抗与 PDN」中的物理含义、单位、边界条件和可测量表现。
- **动态电压容限 $\Delta V$**：理解它在「电源完整性 PI 基础：目标阻抗与 PDN」中的物理含义、单位、边界条件和可测量表现。
- **全频段目标阻抗（$Z_{target}$）计算**：理解它在「电源完整性 PI 基础：目标阻抗与 PDN」中的物理含义、单位、边界条件和可测量表现。

### 2. 关键参数、公式与边界条件

- **核心技术边界**：瞬态负载电流阶跃 $dI$、动态电压容限 $\Delta V$、全频段目标阻抗（$Z_{target}$）计算。
- 电流定义：$I=\frac{dQ}{dt}$；工程判断不只看电流大小，还要看方向、回流路径和变化率。
- 节点电压是相对参考点的电位差；地弹常由公共阻抗上的 $R\cdot I$ 和 $L\cdot\frac{di}{dt}$ 叠加产生。
- 数据转换系统中 $LSB=\frac{V_{ref}}{2^N}$；实际有效精度要由噪声、INL/DNL、参考源和前端驱动能力决定。
- 高速 PCB 判断重点是参考平面连续、阻抗一致、回流路径短、过孔 Stub 可控和 PDN 无尖锐反谐振。
- 传输线反射系数 $\Gamma=\frac{Z_L-Z_0}{Z_L+Z_0}$。
- 目标阻抗 $Z_{target}=\frac{\Delta V}{\Delta I}$ 用于约束 PDN。

### 3. Worst-Case 精算红线

- 所有关键参数必须按最小值、典型值、最大值三组边界建模，不能只按 Typical 值完成设计。
- 对电压、电流、功率、热阻、带宽、耐压、时序、漏电、寿命或认证等级中的关键项建立一页计算书。
- 本课交付物必须能反向证明设计边界：计算：微处理器高速核心电源全频段目标阻抗需求书
- 能说明「电源完整性 PI 基础：目标阻抗与 PDN」在 PCB 叠层、高速 SI/PI 与量产设计中的工程意义。
- 能把大纲中的「瞬态负载电流阶跃 $dI$、动态电压容限 $\Delta V$、全频段目标阻抗（$Z_{target}$）计算。」拆解为可计算、可测试、可评审的指标。
- 能围绕「计算：微处理器高速核心电源全频段目标阻抗需求书」形成原理图、参数计算、测试记录或评审材料。
- 说清“电源完整性 PI 基础：目标阻抗与 PDN”在产品中的作用、失败后果和典型应用场景。
- 掌握本课涉及的关键参数、边界条件和 Worst-Case 计算思路。
- 能把原理落到原理图、PCB Layout、测试验证和工程 Checklist。

## 二、 工程设计落地：计算：微处理器高速核心电源全频段目标阻抗需求书

### 1. 产品场景与架构位置

- 用于多层 PCB 叠层、阻抗控制、高速总线、电源完整性、大电流和 DFM/DFT。
- 适合从原理图阶段规划叠层、回流路径、阻抗、过孔和生产测试。
- 本课交付物对应的典型场景：计算：微处理器高速核心电源全频段目标阻抗需求书

### 2. 原理图架构设计与参数平衡

- 高速信号必须紧邻连续参考平面，避免跨分割和长 Stub。
- 大电流路径按铜厚、线宽、过孔阵列和温升闭环。
- 去耦电容按频段、位置、回流路径和封装 ESL 组合。
- 将「瞬态负载电流阶跃 $dI$、动态电压容限 $\Delta V$、全频段目标阻抗（$Z_{target}$）计算。」转成原理图注释、器件参数、网名、测试点和评审 Checklist。
- 原理图不要只照抄参考设计，要标注关键阻容、保护器件、测试点和可替代物料。
- PCB 需要检查回流路径、地参考、电源路径、热路径、敏感节点和接口防护位置。
- 对高速、大电流、弱信号或高压隔离场景，要单独列出布局禁区和走线约束。
- 评审时至少回答：这个参数为什么是这个值，这条线为什么这样走，测试点为什么放这里。
- 原理图评审时必须把“为什么选这个参数、为什么这样连接、异常能量从哪里走”讲清楚。
- 关键器件旁边保留可测节点，方便首板 Bring-up、失效复现和量产抽检。

### 3. PCB Layout 硬核约束准则

- 高 di/dt、高 dv/dt、大电流、弱信号、高阻节点必须优先定义回流路径，而不是最后补救走线。
- 敏感节点远离开关节点、接口浪涌泄放路径、继电器/电机/电源热源和板边强干扰区域。
- 对外接口、功率回路、模拟前端和高速数字链路必须分别定义地参考、保护路径、测试点和禁布区域。

## 三、 实验室验证与故障诊断闭环

```
静态初查 [万用表/电源限流]  -->  动态捕获 [示波器/差分探头]  -->  边界注入 [负载/温箱/ESD/浪涌/协议分析]
      |                                  |                                      |
  确认短路/漏电/偏置                  捕获纹波/尖峰/时序/振荡                    验证 Worst-Case 与失效模式
```

### 1. 仪器测试与判据

- 用 TDR、VNA、示波器眼图和电源噪声测试验证阻抗、反射、串扰和 PDN。
- 板厂阻抗 Coupon、叠层参数和实测数据要与设计约束闭环。
- DFM/DFT 检查覆盖焊盘、间距、测试点、拼板、工艺边和可维修性。
- 测试记录至少包含样机版本、仪器型号、连接方式、环境条件、波形截图和判定结论。
- 万用表：用于静态电压、电阻、通断、漏电或短路初查。
- 示波器：用于时序、纹波、尖峰、振荡、启动过程和异常复现。
- 逻辑分析仪/协议分析：用于数字接口、总线时序和协议帧定位。
- 电子负载/电源/热像仪：用于功耗、温升、瞬态响应和保护动作验证。
- 若涉及 EMC/SI/PI，再引入近场探头、LISN、频谱仪、TDR 或 VNA 等工具。
- 每一项测试都必须写清样机版本、测试点、探头连接、带宽限制、负载条件、温度条件和通过/失败判据。

### 2. 典型故障与排查范式

- 信号线跨参考平面分割，回流绕路导致 EMI 和眼图劣化。
- 过孔 Stub 或反焊盘处理不当，高速链路出现频率选择性反射。
- PDN 反谐振峰未控制，某些负载瞬态下芯片复位或误码。
- 排查时按「现象 → 假设 → 测量 → 结论 → 预防」的顺序闭环，不跳过证据链。
- 参数只按典型值设计，没有覆盖最大/最小值、温度、批次离散和老化。
- 原理图正确但 PCB 回流路径错误，导致噪声、误触发、通信失败或采样漂移。
- 测试方法不规范，例如示波器长地线把纹波、尖峰和 EMI 测错。
- 保护电路只考虑能不能钳位，没有核算能量、热、响应时间和失效模式。
- 只验证单点功能，没有做边界、长时间、温度、负载和异常恢复测试。
- 排查必须遵循“现象 -> 假设 -> 测量 -> 结论 -> 预防 Checklist”的证据链，不允许只凭经验改板。
