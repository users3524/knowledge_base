# 嵌入式软件工程师 150 堂高质量课程大纲

> 定位：面向嵌入式软件、MCU 裸机、RTOS、Linux 应用、Linux 驱动、通信协议、物联网软件与就业项目方向。  
> 授课原则：每堂课都围绕“为什么学、底层原理、实战代码、调试方法、常见坑、面试表达、课后任务”展开。  
> 课程目标：让学员具备 MCU 裸机开发、RTOS 项目开发、Linux 应用开发、Linux 驱动开发、通信协议、物联网项目与面试就业能力。

## 课程总览

| 模块 | 课次范围 | 核心能力 |
|---|---:|---|
| 第一阶段：嵌入式认知与开发环境 | 1-10 | 建立学习路线、开发环境、工具链和工程意识 |
| 第二阶段：C 语言实战基础 | 11-35 | 掌握嵌入式 C 语言、指针、内存、结构体和工程化 |
| 第三阶段：数据结构与嵌入式工程能力 | 36-45 | 链表、队列、状态机、日志、模块化和调试能力 |
| 第四阶段：STM32 裸机开发 | 46-75 | GPIO、中断、定时器、串口、ADC、I2C、SPI、DMA、低功耗 |
| 第五阶段：FreeRTOS 实时操作系统 | 76-95 | 任务、队列、信号量、互斥锁、事件组、内存和综合项目 |
| 第六阶段：Linux 基础与系统编程 | 96-115 | Linux 命令、Makefile、Git、文件 IO、进程线程、网络编程 |
| 第七阶段：ARM Linux 移植与驱动开发 | 116-135 | U-Boot、Kernel、设备树、字符设备、platform、I2C/SPI 驱动 |
| 第八阶段：通信协议、物联网与就业项目 | 136-150 | Modbus、CAN、MQTT、OTA、工业网关、面试和项目包装 |

## 每堂课的高质量授课标准

1. **先讲应用场景**：明确这个知识点在产品中解决什么问题。
2. **再讲底层原理**：不只教 API，要讲寄存器、内存、时序、系统调用或内核机制。
3. **必须上手实战**：每堂课至少有一个可运行实验或可验证产出。
4. **必须讲调试方法**：串口日志、断点、寄存器观察、逻辑分析仪、dmesg、gdb 等按场景使用。
5. **必须讲常见坑**：例如中断里不能做耗时操作、DMA 缓存越界、I2C 无 ACK、线程死锁、设备树匹配失败等。
6. **必须能转成面试表达**：课后让学员能用 1-3 分钟讲清楚核心原理。
7. **必须服务项目闭环**：所有知识最终进入 MCU 项目、RTOS 项目、Linux 应用项目或 Linux 驱动项目。
