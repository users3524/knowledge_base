# Materials Audit

Goal: inspect the user's raw materials and identify usable evidence.

## What To Extract

- Projects.
- Internships.
- Code evidence.
- Metrics.
- Bad cases.
- Logs / screenshots.
- Papers read.
- Competitions / awards.
- Coursework.
- Open-source contributions.
- Ownership boundaries.

## Evidence Levels

| Level | Meaning | Examples |
| --- | --- | --- |
| strong | Can support a resume claim | code diff, report, metric table, log, README, demo screenshot |
| medium | Useful but needs explanation | notes, course project, partial experiment, small comparison |
| weak | Cannot support strong wording alone | vague memory, final screenshot, copied repo, no run record |
| missing | Needed but absent | no metric, no ownership, no runnable proof |

## Audit Rules

- Separate personal work from team/platform/open-source work.
- Mark unsupported metrics as missing evidence.
- Mark production claims as risky unless there are logs, releases, users, or internal trial evidence.
- For code folders, look for README, tests, scripts, examples, commits, notebooks, logs.
- For PDFs, prefer text exports; if unreadable, ask for converted text.
