# Intake Questions

Ask only the missing questions. Do not require the user to answer everything if the files already contain enough evidence.

## Must-Have

1. What is the target JD or target role?
2. Where is the current resume?
3. Which project or experience should be emphasized?
4. What did you personally do?
5. What is the project status: production, internal trial, demo, course project, local run, or not runnable?
6. What evidence exists: code, logs, screenshots, metrics, reports, bad cases, README, or commits?

## Role-Specific

For RAG / Agent / LLM app:

- Did you build retrieval, rerank, citation, evaluation, guardrails, or only prompts?
- What are the input, output, failure cases, and evaluation method?
- Is there trace logging, retry, schema validation, or human review?

For search / ranking:

- Do you understand recall, rough rank, fine rank, rerank, query-doc features, and ranking metrics?
- Is there any search/recommendation/ranking project, contest, or course evidence?
- Did you do query analysis, doc understanding, long-tail cases, or relevance evaluation?

For LLM algorithm:

- Did you train, fine-tune, evaluate, or only call APIs?
- What model, dataset, objective, metric, and baseline did you use?
- Can you explain Transformer, SFT, LoRA, DPO/RLHF, evals, or inference optimization?

## If Evidence Is Weak

Ask:

- How much time can you spend: half day, 1 day, 3 days, or 1 week?
- What hardware do you have?
- Are you willing to reproduce or modify an open-source project?
- Can you publish a personal repo, README, screenshots, logs, or report?
